# EPIC: Landing Promotion & Distribution

## Goal
Promote the public documentation platform to developers and recruiters
without paid ads or aggressive marketing.

## Promotion Channels (FIXED)

### GitHub
- Public repository visibility
- Clear README with project purpose
- Link to live docs in repository description

### LinkedIn
- One announcement post about the project
- Focus on:
  - GraphQL learning
  - API design
  - Developer tooling
- Link to live docs

### Telegram
- Share link in developer-focused chats or channels
- Short description + value proposition

### Direct Sharing
- Use as portfolio / showcase project
- Share link during interviews or code reviews

## Tracking
- All promotion traffic must be measurable via Umami
- Referrers must be visible in analytics dashboard

## Rules
- No paid advertising
- No spam or mass-posting
- No SEO keyword stuffing
- No growth hacks

## Acceptance Criteria
- Landing is publicly accessible
- Link can be safely shared externally
- Traffic sources are visible in analytics
- Project is suitable as a portfolio reference
